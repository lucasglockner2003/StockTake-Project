# SmartOps StockTake System

This project was created to solve a real operational problem in large restaurant kitchens.

Stocktaking usually takes up to 6–7 hours and is prone to human errors because it is done on paper and manually transferred to a computer.

This system helps automate and speed up the process.

## Features

- Real-time stock status alerts
- Critical / Low / Check indicators
- Smart grouping by kitchen area
- Fast input interface for high-pressure environments
- Local storage auto-save

## Tech Stack

- React
- Vite
- JavaScript
- LocalStorage

## Future Improvements

- Voice input for stock counting
- Database integration
